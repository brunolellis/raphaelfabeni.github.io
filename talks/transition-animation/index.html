Transition

Adiciona um efeito quando o navegador troca de um estilo para outro, sem usar Flash ou animaçoes em JavaScript.

O que já fizemos e exemplos

Como faz essa joça?

São quatro propriedades:

transition-property Especifica o nome da propriedade CSS a qual a transição vai ser aplicada.
transition-duration Duração do efeito.
transition-timing-function Forma como a transição progride no tempo.
transition-delay Define a partir de quanto tempo (em segundos) o efeito vai iniciar (Padrão é 0).

Resumindo


               div {
                  transition-property: width;
                  transition-duration: 1s;
                  transition-timing-function: linear;
                  transition-delay: 2s;
               }

Animation

Com as animações em CSS3, podemos criar animações que podem substituir imagens animadas e animações em Flash ou JavaScript em várias páginas.

Exemplos

Eaí, faz como?

A Regra dos Keyframes

É aonde as animações são criadas.

Cada um contém uma propriedade CSS e a animação se encarrega de mudar de um keyframe para outro.


               @keyframes aegaroto {
                  from {background: red;}
                  to {background: yellow;}
               }

O bom é que esse não tem prefixo né...

                  to { background: yellow; }

Continuando a mágica...

Com os keyframes criados precisamos vinculá-los a algum seletor, senão a animação não acontece.

Pra isso precisamos de duas coisas:

  • O nome da animação (igual ao especificado na chamada dos keyframes)
  • A duração da animação (se não for declarada, a animação não se inicia pois o valor padrão é 0)

               div {
                  -webkit-animation: aegaroto 5s;                  
                     -moz-animation: aegaroto 5s;
                       -o-animation: aegaroto 5s;
                      -ms-animation: aegaroto 5s;
                          animation: aegaroto 5s;
               }

As propriedades

animation-name Nome da animação especificada nos @keyframes.
animation-duration Quanto tempo durará um ciclo da animação (Padrão é 0).
animation-timing-function Como a animação vai avançar durante um ciclo (Padrão é ease).
animation-delay Determina quando a animação se inicia (Padrão é 0).
animation-iteration-count Número de vezes que a animação vai se repetir (Padrão é 1).
animation-direction Se ao final da animação, ela deve reiniciar seu fluxo normalmente, ou voltar no sentido inverso (Padrão é normal).
animation-play-state Se a animação está rodando ou pausada (Padrão é running).
